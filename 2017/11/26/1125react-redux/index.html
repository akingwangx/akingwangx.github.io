React-Redux初窥

发表于 2017-11-26

前言

几天前实现了项目功能的需求后就开始学习Redux，开头两日很是痛苦，看的很懵，冷静的想了很久才稍微想通了一点，本篇文章主要讲一下自己理解了的React-Redux，至于前置的知识flux，redux什么的就不展开了，一搜一大推(虽然React-Redux也是。。)，建议知乎上搜下redux，大佬们举的例子还是有助于理解的，现在我们来开始学习React-Redux吧。

想放在最开始讲的是redux里有个比较重要的函数：reducer。它主要负责接收两个参数，第一个参数为数据(state初始状态)，第二个参数为action操作对象，reducer要return的是新的state，而不能直接在原有state上面做修改~

一个reducer：

let reducer-wx=function(state=0,action){
  switch(action.type){
    case:'一个action':
       return state+action.value;
    default
       return state;
  }
}

刚看到这个reducer我也是很不理解它是干嘛的，以为又是得死记的API，记住怎么用什么的，结果那一天又是store、dispatch、createStore、action、reducer的记得记得就混乱了，效率低的过完了一天，第二天查了资料发现自己还是js基础不牢靠，有一个Array.protoType.reducer方法自己竟然都不知道，然后看完之后发现稍微能理解一点redux里的reducer函数了。

使用React-Redux

React-Redux在我理解就是提供了两个很有用的方法，一个是Provider，一个是connect，先来讲Prvoider，我理解的很简单，就是用 < Proviider>标签将整个React.Dom要渲染的最外层组件包裹起来，然后属性上加上store属性，让所有子组件可以用this.props接收到redux里store的那些方法。

connect就稍复杂一点，他看起来是这个样子的：


let App=connect(接受参数完成配置阶段，包裹成一个新东西后跟后面的组件建立联系)(要连接的组件)

connect(a,b)(component)中a参数用来接受store中的数据，通过props传给React组件(一个具体的state数据)，b参数用来接受action发出的那个定义的动作，通常是const一个全大写的名字然后通过函数返回

然后这个connect可以写成装饰器的形式我觉得很简洁懂了的话也好看

@connect(state=>({num:state}),
         {b参数}
)
class App extends React.Component{
  ...
  这里如果有用到事件的话用this.props.b参数名来接收b那个action，等于用Provider省去了dispatch，直接就可以用props接收到action
  例：<Button onClick={this.props.addState}>yo</Button>
}

以上就是我对React-Redux的浅显理解，1点半了眼睛有点睁不开，就先睡了，下周实践后再做总结吧，这个写的有点辣眼睛。。
