发表于 2017-10-19

今天是实习的第一天，也是真的开始学React技术栈的第一天，之后可能每一天都会写一点东西总结一下，感觉公司每个人都干劲十足，很害怕实习期过了没跟上脚步被淘汰呢，话不多说，开始~

import

import React from 'react'；
import ReactDOM from 'react-dom';

从头开始看，首先是在 JS 当中引用其他库文件的语法变化。之前我们一般都是通过 require 方法把库文件导出的方法保存在一个变量中。在 ES6 当中引入了一组个新的关键字 import/export，在文件的开头引入我们需要使用的模块或方法。

我的理解是import引用的是整个的JS文件，而export则引用的是单个的函数。

然后是ES6的let和const方法，从第一天的学习来看const用的很多，基本每次组件定义常量都会使用，let暂时还没有用到。

然后是class，在React之前的版本中，多使用createClass的方法来声明一个组件，不过目前好像还得加别的包才可以使用，导致我找了半天视频都是旧的使用这个方法的，浪费了很多时间。

提取组件： 可以将组件分为更小的组件

提取组件一开始看起来像是一项单调乏味的工作，但是在大型应用中，构建可复用的组件完全是值得的。当你的UI中有一部分重复使用了好几次（比如，Button、Panel、Avatar），或者其自身就足够复杂（比如，App、FeedStory、Comment），类似这些都是抽象成一个可复用组件的绝佳选择，这也是一个比较好的做法

function forcat(date){
  return date.toLocaleDateString();
}
function Comment(props){
  return(
    <div className="Comment">
      <div className="UserInfo">
        <img src={props.aut.url} alt={props.aut.name} className="AV"/>
       <div className="UserName">{props.aut.name}</div>  
      </div>
      <div className="comment-text">{props.text}</div>
      <div className="comment-date">{forcat(props.date)}</div>

    </div>
  )
}
const author={
  date:new Date(),
  text:"I hope you are happy!",
  aut:{
    name:"cat",
    url:"http://placekitten.com/g/64/64"
  }
}
ReactDOM.render(
  <div>
    <Comment date={author.date} text={author.text} aut={author.aut}/>
  </div>,
  document.getElementById("root")
)
